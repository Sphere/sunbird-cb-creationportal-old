## Summary

A patch release on top of 5.0.0 with two content-authoring bug fixes — the **"Show Correct Answer Popup"** option now appears for **Assessment** resources only (it was leaking onto PDFs/other types), and the course **Description** is now capped so an over-long, heavily-formatted paste can no longer break content search indexing. The rest is developer-workflow hardening: unit tests are now a CI merge gate, the branching/release process is codified, and the stale dev/preprod Docker images were modernised.

## ✨ Features

- _None — patch release._

## 🐛 Fixes

- **author / course builder** — **"Show Correct Answer Popup" now shows for Assessment resources only**, never Quiz, Self Assessment, or PDF/audio/video/link. It was gated only on `isAssessment` (which can leak truthy onto other types); now gated on a precise `isAssessmentResource` check (assessment mimeType + `isAssessment` + not self-assessment), and the save paths are hardened with the same guard (`340cb78b`).
- **author / course details** — **course Description (`instructions`) is capped at 24,000 UTF-8 bytes** via a reusable `maxByteLengthValidator`, with a live byte counter and a clear over-limit message. A Hindi + Word/Docs-pasted description had reached ~34,939 bytes, exceeding Elasticsearch's 32,766-byte keyword-term limit and crash-looping the composite search indexer — which stopped new content from being indexed (newly created courses didn't appear in Drafts). The cap is byte-based (multibyte-safe) with headroom for markup (`e5539571`, `bb8d28d4`).

## 🏗️ Build / CI / Infra

- **ci** — the GitHub Actions **Quality** workflow now runs **Jest with the coverage threshold** as a hard gate (was lint + build only), so tests/coverage can't regress on a PR (`2d22fcf1`).
- **docker** — modernised stale `Dockerfile.dev` / `Dockerfile.preprod` (`node:10` → `node:20` multi-stage; dropped removed Angular-8 build flags) to match the production image (`2d22fcf1`).

## Release & rollback

**Deploy** — a human runs the manual Jenkins job (`Jenkinsfile-sun`) with `github_release_tag = cbp-release-5.0.1` (the **tag**). Each patch on the 5.0.x line ships as its own immutable tag — the `cbp-release-5.0` branch stays frozen at the 5.0.0 snapshot and is **not** advanced per patch.

**Rollback** — re-run the same manual Jenkins job against the previous release ref:

```text
github_release_tag = cbp-release-5.0.0
```

(Jenkins → Docker → Helm pipeline.)
