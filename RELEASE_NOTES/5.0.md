## Summary

This is a **major platform release**: the Creation Portal has been migrated from **Angular 8 (ViewEngine) to Angular 21** (esbuild builder, served by an Express runtime), with Keycloak and Angular Material upgraded along the way. On top of the migration, the authoring experience was rebuilt on a unified LMS design system and the course creation, review, and publish workflows were hardened — step-by-step validation, role-aware dashboards for reviewers/publishers, and accessible, on-brand dialogs. Content authors get a faster, more consistent, and more guided experience; the rest of the change is the framework upgrade itself.

> The bulk of the 892 commits is the migration history across four milestones (8→12→15→17→21). The sections below highlight the meaningful, user- and ops-facing changes; every bullet is traceable to a commit.

## ⚠️ Deploy notes & risk

> This is the first prod deploy of the Angular 21 migration — all three migration traps are in play. Read these.

- **Migration/deploy gotchas touched?**
  - [x] `outputPath` / `dist/www/en` layout (Express prod server, **not nginx**) — output must stay `base: dist/www/en`, `browser: ""`, or root 404s.
  - [x] `ckeditor4-angular` must stay **≥ 5.2.1** (ngcc/Ivy trap) — a green *local* build is not proof; validate on a **fresh install**.
  - [x] Build flags — kebab-case `--base-href=/` only; do **not** add `--output-path` on the CLI.
- **Config / env / secret changes:** Keycloak provider wiring changed (`provideKeycloak` → direct `KeycloakService`); verify the realm/redirect config in the target env.
- **Backend / API contract dependencies:** none changed — same `/apis/proxies/v8/` gateway.
- **Breaking changes:** runtime is now **Express** (`dist/server.js` via `npm run serve:prod`), not nginx — update the serving layer / container entrypoint accordingly.

## ✅ Pre-deploy checklist
