## Summary

This is a **major platform release**: the Creation Portal has been migrated from **Angular 8 (ViewEngine) to Angular 21** (esbuild builder, served by an Express runtime), with Keycloak and Angular Material upgraded along the way. On top of the migration, the authoring experience was rebuilt on a unified LMS design system and the course creation, review, and publish workflows were hardened — step-by-step validation, role-aware dashboards for reviewers/publishers, and accessible, on-brand dialogs. Content authors get a faster, more consistent, and more guided experience; the rest of the change is the framework upgrade itself.

> The bulk of the 900 commits is the migration history across four milestones (8→12→15→17→21). The sections below highlight the meaningful, user- and ops-facing changes; every bullet is traceable to a commit.

## ✨ Features

- **Platform** — migrated the entire app to **Angular 21** with the modern `@angular/build:application` (esbuild) builder.
- **create** — unified LMS design system across the full course-creation workflow, with the step "Next" delegated to a parent stepper (`d5b47584`, `2015880d`).
- **design system** — applied the unified design system across all components, the self-assessment form, and the TOC / org / author / my-content modules (`e5543bda`, `ee3e6880`, `105e7b02`).
- **create** — redesigned footer, stepper, and error dialog, and fixed the course-creation flow (`d9ff4f9f`).
- **create** — Course Details / Builder / Settings now gate the stepper **Next** until each step is valid (required fields, ≥ 2 resources) (`0acf620c`).
- **author** — creators can **download course resources** — a single resource (file as-is, or a quiz exported to Excel) or the whole course as one ordered `.zip`; available in the course builder, the TOC banner, and per-resource cards (`a4ed89d6`).

## ⚠️ Deploy notes & risk

> This is the first prod deploy of the Angular 21 migration — all three migration traps are in play. Read these.

- **Migration/deploy gotchas touched?**
  - [x] `outputPath` / `dist/www/en` layout (Express prod server, **not nginx**) — output must stay `base: dist/www/en`, `browser: ""`, or root 404s.
  - [x] `ckeditor4-angular` must stay **≥ 5.2.1** (ngcc/Ivy trap) — a green _local_ build is not proof; validate on a **fresh install**.
  - [x] Build flags — kebab-case `--base-href=/` only; do **not** add `--output-path` on the CLI.
- **Config / env / secret changes:** Keycloak provider wiring changed (`provideKeycloak` → direct `KeycloakService`); verify the realm/redirect config in the target env.
- **Backend / API contract dependencies:** none changed — same `/apis/proxies/v8/` gateway.
- **Breaking changes:** runtime is now **Express** (`dist/server.js` via `npm run serve:prod`), not nginx — update the serving layer / container entrypoint accordingly.

## ✅ Pre-deploy checklist

## Release & rollback

**Deploy** — this release ships as the **`cbp-release-5.0`** branch; a human runs the
manual Jenkins job (`Jenkinsfile-sun`) with `github_release_tag = cbp-release-5.0`. The
branch is just the deploy source — pushing it does not deploy on its own.

**Rollback** — re-run the same manual Jenkins job against the previous release ref:

```text
github_release_tag = cbp-release-4.0   # the prior prod release
```

(Jenkins → Docker → Helm pipeline, namespace `dev`.)
